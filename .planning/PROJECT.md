# Зеркало Судьбы — Telegram Mini App для AI-раскладов таро

## What This Is

Telegram Mini App для атмосферных раскладов таро с генеративной интерпретацией. Пользователь задаёт вопрос, выбирает тему, колоду и расклад — и получает красивый интерактивный ритуал с глубокой, но бережной интерпретацией каждой карты и общим итогом. Внутри интерпретации работает LLM, но в UI продукт воспринимается как «цифровая гадалка / оракул», а не как AI-сервис: слова «AI», «нейросеть», «модель», «сгенерировано ИИ» в интерфейсе не используются.

Аудитория: 18–35 лет, интерес к таро, саморефлексии, отношениям, мистическому контенту и эстетичным mini apps; пользователи Telegram, которым нужен быстрый красивый расклад без установки отдельного приложения.

## Core Value

**Один и тот же вопрос ощущается по-разному в разных колодах.** Если ритуал, атмосфера и персональность ответа работают — продукт живёт; всё остальное вторично. Это ядро MVP (ТЗ §30): 6 колод с разными prompt-модификаторами, тоном и визуалом дают разный опыт на один вопрос.

## Context

**Технический контекст.** Greenfield-проект, монорепо: `/frontend`, `/backend`, `/bot`, `/admin` (как защищённые роуты во frontend), `/docs`. Локальная разработка через Docker Compose (Postgres + Redis). Деплой на timeweb.cloud.

**Продуктовые принципы (ТЗ §1.5).** Не «AI chat», а цифровой ритуал — пользователь обращается к колоде, не к боту. Не предсказание-приговор, а подсказка; никаких категоричных обещаний будущего. Каждая колода меняет опыт (картинка + тон + структура ответа). Меньше сухого текста, больше атмосферы. Без запугивания и давления, особенно в темах отношений/денег/здоровья/тревоги. Быстрое первое впечатление — первый расклад без длинной регистрации.

**Авторские права на колоды (ТЗ §5).** Нельзя: копировать реальные известные колоды, сканы, композиции «один-в-один», названия коммерческих колод в UI, ассеты из Pinterest/Google без лицензии, генерацию «in style of [living artist/deck]». Можно: общая структура таро (старшие/младшие арканы, масти, архетипы), авторские стили, свои названия. Универсальное значение карты хранится отдельно от стилевого слоя колоды (модель данных: `cards` + `deck_cards`).

**Безопасность контента (ТЗ §20).** Приложение не даёт мед./юр./фин. категоричных советов, не гарантирует доход, не утверждает чувства другого человека как факт, не запугивает, не использует фаталистичные формулировки. Чувствительные темы (здоровье, беременность, смерть, самоповреждение, насилие, юр. риски, кредиты, инвестиции, измена, зависимые отношения) обрабатываются особенно осторожно. Кризис → не мистическое предсказание, а поддержка + предложение обратиться к живому человеку/службам.

**Исходное ТЗ.** Полный 60-страничный PRD сохранён в `.planning/REFERENCE-TZ.md` (источник истины: модель данных, API, prompt-шаблоны, палитры, seed-data, этапы).

## Constraints

- **Tech stack (frontend)**: React + TypeScript + Vite + Tailwind CSS + Framer Motion + **Zustand** (не Redux — легче для mini app) + React Query + Telegram WebApp SDK — задано ТЗ §12.1 с уточнением стейта (моя правка #5)
- **Tech stack (backend)**: FastAPI + Python 3.12+ + SQLAlchemy 2.x async + Alembic + Pydantic + PostgreSQL + Redis (rate-limit/кэш) — ТЗ §12.1; **без Celery/RQ/Arq** (моя правка #2)
- **Tech stack (bot)**: aiogram 3.x, webhook mode, обработка payments updates — ТЗ §12.1
- **LLM**: абстракция `LLMService` (swappable provider), дефолт Claude; **один структурированный вызов на весь расклад** (карты + итог в одном JSON), per-card шаблоны как fallback — стоимость/латентность (моя правка #1)
- **Payments**: только Telegram Stars (валюта XTR) — требование Telegram для цифровых услуг; доступ засчитывается только после `successful_payment`, идемпотентность по `payload`
- **Safety**: классификатор обязателен (поднят из «желательно» ТЗ §20.4 — моя правка #3); запрет категоричных предсказаний/мед.-юр.-фин. советов; кризис → безопасный поддерживающий ответ
- **Legal/IP**: только авторские визуалы; юридическая проверка ассетов перед публичным запуском
- **Deploy**: timeweb.cloud, HTTPS обязателен. MCP-доступ ограничен (только App Platform deploy из Git; managed PostgreSQL/Redis/S3 и VPS — вручную в панели или контейнерами) — деталь развёртывания решается на этапе деплоя
- **Brand voice**: в UI/текстах/кнопках/результатах не использовать «AI», «нейросеть», «модель», «сгенерировано ИИ» (кроме юридически необходимых мест)
- **Mobile-first**: основной дизайн под смартфон 360–430px; нижняя sticky-CTA; крупные карты
