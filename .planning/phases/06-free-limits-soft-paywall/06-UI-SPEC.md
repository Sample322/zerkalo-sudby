**Rules (inherited + phase-specific):**
- Only weights 400 and 600. Hierarchy comes from size + the `--deck-soft` opacity ladder (full-opacity headline, `opacity-70` count line, `opacity-60` tertiary), matching the existing selection screen.
- The selection-screen remaining-count is a **Label** line at `text-sm` `opacity-70` — deliberately quiet so it never competes with «Начать расклад» (D-09 "subtly"). It sits styled exactly like the existing `START_GATE_HINT` paragraph (`px-1 ... text-sm opacity-70`).

- Paywall body copy: max ~2 short lines per block, separated by `lg` (24px). No walls of text — the paywall must feel light and honest, not a sales page.

### Interpolation contracts (component composes; helper lives in `copy.ts` so it is ban-list scanned)

- **Remaining count** (selection + profile): render `Осталось {free_left} из {free_weekly_limit}` where `free_left = limits.free_weekly_limit − limits.free_used_this_week` (clamped ≥ 0) and `total = limits.free_weekly_limit`, both from `GET /api/me` `limits` (`useMe`). Provide a pure helper, e.g. `formatRemaining(left: number, total: number): string`. When `free_left === 1`, the component ALSO shows `LIMIT_LAST_ONE_HINT` (the gentle accent line, D-10). When `free_left === 0`, the count line is suppressed on the selection screen (the paywall carries the state instead) but the profile may still show «Осталось 0 из 3».
- **Reset countdown** (paywall, D-04): the backend's limit-block response is the source of the reset moment. Provide a pure helper `formatReset(resetAt: string | Date, now?: Date): string` that returns a brand-safe phrase:
  - if the reset is within ~48h → relative days: «через 1 день» / «через 2 дня» / «через N дней» (Russian plural rules — `день` / `дня` / `дней`),
  - else → an absolute date: «16 июня» (Russian month genitive).
  The paywall renders `PAYWALL_RESET_LEAD + formatReset(resetAt)` (e.g. «Бесплатные расклады вернутся через 2 дня» / «… вернутся 16 июня»). The reset-moment value comes from the API limit-block body (planner defines the exact field; D-01 makes it per-user). If the value is absent for any reason, fall back to a non-numeric reassurance «совсем скоро» — never show a broken/empty countdown, never «NaN».

## Screen & Interaction Contracts

No new routes / no new `Step` (consistent with the Phase-5 nav decisions and D-05). All three surfaces are STATE on existing screens. The error source is the `createReading` → `POST /api/readings` mutation (429 / limit-block / honest-fail) and the `GET /api/me` `limits` read.

### Error-branch contract (the seam that splits throttle vs paywall vs failure)

`createReading.ts` currently rejects with a generic `Error` on any non-OK status or non-completed body. Phase 6 must make the rejection **discriminable** so `CatalogScreen.handleStart`'s `catch` can route to the correct surface (D-08 — never conflate). The contract:

| Backend signal | Discriminant the FE branches on | Surface shown | Limit consumed? |
|----------------|--------------------------------|---------------|-----------------|
| HTTP 429 (Redis throttle, D-07) — FIRST gate | a typed error carrying `kind: "throttle"` (e.g. `status === 429`) | **Throttle toast** (transient, auto-dismiss) — `THROTTLE_MESSAGE` | No (never reached PG) |
| Limit-block body (free exhausted, D-03) — soft 200 or a dedicated status the planner picks | a typed error carrying `kind: "paywall"` + the reset moment | **Paywall bottom-sheet** (persistent until dismissed) — `PAYWALL_*` | No (D-09) |
| Honest-fail / safety / non-completed (Phase-4 D-08) | the existing non-completed branch (`kind: "failure"` / default) | **Existing failure band** — `READING_ERROR` + retry/change-deck | No (D-09) |

Planner decides the exact transport (a thrown typed `ReadingError` subclass vs a discriminated return) and the backend status/shape for the limit block; the UI contract is only that these three are **visually and behaviorally distinct** and routed from one `catch`. `createReading` keeps its existing signature (D-05/D-07 architecture guard) — extend the error type, not the return type.

### Surface 1 — Soft paywall (D-03, D-04, D-05) — on `CatalogScreen.tsx`

- **Form:** a **bottom-sheet** anchored to the existing fixed bottom band region (NOT a dedicated route, NOT a full-screen takeover — TZ §9.7 dedicated tariffs screen is deferred to Phase 7, D-05). It renders in the same `fixed inset-x-0 bottom-0` zone the «Начать расклад» band occupies, OR as an in-place panel that replaces the CTA band content (planner's call; both satisfy "inline/sheet on the selection screen"). A dimmed scrim (`--deck-bg` ~70%, animate `opacity`) sits between the sheet and the scrollable selection content to focus attention without hiding the screen.
- **Trigger:** when `free_left === 0` (known from `useMe` `limits`), tapping «Начать расклад» opens the paywall sheet **instead of** starting a reading. Belt-and-suspenders: if the user somehow reaches `POST /api/readings` and the backend returns the limit-block, the `catch` opens the same sheet (the `kind: "paywall"` branch). The CTA itself is NOT replaced by a "buy" button — it stays «Начать расклад»; the gate is the sheet.
- **Contents (top → bottom), each block separated by `lg` (24px):**
  1. `PAYWALL_TITLE` (Display 28px, `--deck-soft`).
  2. `PAYWALL_RESET_LEAD` + `formatReset(resetAt)` (Body 16px; the reset value span tinted `--deck-accent` — the single hopeful focal point, D-04).
  3. `PAYWALL_SOON_NOTE` (Body 16px, `opacity-70` — quiet, no pressure, D-03).
  4. A dismiss / back affordance: a small accent-tinted glyph button (reuse the `h-10 w-10` icon-button pattern, hit-area padded to ≥44px) labelled `PAYWALL_DISMISS`, OR tapping the scrim. Dismissing returns to the live selection screen with question + selections **preserved** (never clear `question` — mirrors the existing `handleChangeDeck` D-04 behavior).
- **What it must NOT contain:** tariffs, prices, Stars, any "купи"/"buy"/"оплатить" button, any dead CTA, any countdown timer that ticks (a static "через N дней"/"date" phrase is correct — no live ticking clock), any fear/urgency framing.
- **Motion (inherited Phase-3 language):** sheet enters via `m.div` `initial {opacity:0, y:24}` → `animate {opacity:1, y:0}`, exit `{opacity:0, y:24}`, `transition {duration: 0.28, ease: [0.16, 1, 0.3, 1]}` (the locked screen-enter token). Scrim crossfades `opacity` 0→1 over the same 0.28s. Compositor-only (`transform`/`opacity`) — banned: animating `height`, `top`, `margin`, `box-shadow` blur. Use `AnimatePresence` to mount/unmount the sheet. Optional `haptic.notify("warning")` is **discouraged** here (it implies alarm — brand); if any haptic, a soft `haptic.selection()` on open is acceptable.

### Surface 2 — Throttle message (D-08) — on `CatalogScreen.tsx`, DISTINCT from the paywall

- **Form:** a **transient toast / inline ephemeral line** — NOT the persistent sheet. It is visually and behaviorally different from the paywall (D-08: never conflate the transient throttle with weekly exhaustion). Recommended: a small glass pill/toast that appears near the CTA band (or top of the safe area) and **auto-dismisses after ~3.5–4s** (reuse the self-contained `setTimeout` + `AnimatePresence` pattern already proven in `UndoSnackbar.tsx` — no toast library, zero new dependency). - **Motion:** enter `{opacity:0, y:8}` → `{opacity:1, y:0}`, exit `{opacity:0, y:-8}` (or fade), `duration 0.28 ease-out-expo`. Compositor-only. `haptic.notify("warning")` is acceptable here ONLY as a very soft "slow down" cue, or omit entirely (planner's discretion — lean toward omitting to stay gentle).

## Anti-Template Compliance (web/design-quality)

These additive surfaces must not introduce generic template patterns and must demonstrate ≥4 required qualities. Declared intent:
1. **Hierarchy via scale contrast** — paywall uses the 28/16 ladder + the `--deck-soft` opacity steps; the count line is deliberately the quietest element (Label, `opacity-70`), not uniform emphasis.
2. **Intentional rhythm** — paywall blocks separated by `lg` (24px), not uniform padding; the count line is spatially subordinate to the CTA.
3. **Depth/layering** — the paywall is a glass sheet over a dimmed scrim over the live deck canvas (real layering); the toast is a glass pill above the surface.
4. **Color used semantically** — accent is rationed to the ONE hopeful focal point (the reset moment) and the single 1-remaining hint; the absence of any red/alarm hue is itself a deliberate brand statement (no fear).
