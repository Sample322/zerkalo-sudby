## Trust Boundaries

| Boundary | Description |

| concurrent POST /api/readings → user_limits UPDATE | Two requests at the boundary must not both consume a slot |
| create_reading early exits → user_limits counter | Crisis/abusive/honest-fail must leave the net counter correct (READ-10) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Tampering / Elevation | the free-quota check+decrement (the boundary race) | mitigate | Single conditional `UPDATE … WHERE free_used < limit … RETURNING` — PostgreSQL's row lock serializes concurrent writers; exactly one matches and increments, the other matches zero rows. THE security control for success-criterion 3 (Task 2/3), proven by `test_limit_concurrency` (mutation-tested: breaking the WHERE makes it observe used==4) |
| T-06-06 | Tampering | the rolling reset (separate-call double-spend) | mitigate | The reset is folded into the SAME atomic UPDATE via `case()` — no read-then-reset-then-decrement window where two requests both reset and both increment (Pattern 2, Task 2) |
| T-06-07 | Elevation / abuse | over-charge or under-charge on non-success exits | mitigate | Safety short-circuits BEFORE the consume (crisis/abusive never consume); the only post-consume exit (honest-fail) refunds — net counter correct, READ-10 preserved (Pitfall 2, Task 3); regression-fenced by the four `test_limit_untouched_on_*` tests |
| T-06-08 | Information Disclosure | the limit-block body | mitigate | The soft body carries only the caller's own `remaining`/`reset_at` (their own reopen time) — no cross-user fields, no SQL, no stacktrace (`_soft_body` is a deliberate 200, the global handler stays the 500 path) |
| T-06-09 | Tampering | SQL injection in the new UPDATE/case() | mitigate | Parameterized SQLAlchemy Core (`update()`/`case()`/bind params) — no string-built SQL (same discipline as the existing `on_conflict_do_update`) |
| T-06-SC | Tampering | npm/pip installs | mitigate | Zero new packages (RESEARCH Standard Stack); none added by this plan |
