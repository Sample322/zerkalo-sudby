<objective>
Add the Redis throttle as GATE 0 of `POST /api/readings` — an atomic Lua `INCR` + conditional-`EXPIRE` per-user counter that rejects bursts with a retryable 429 before any Postgres session opens or any LLM call is made (LIMIT-05, success-criterion 4). Each reading is a paid LLM call, so capping the burst at the cheapest possible layer is the anti-abuse / DoS mitigation.

Purpose: the throttle must run "before it reaches Postgres" (TZ §29.2). A plain `INCR` then `EXPIRE` as two awaits has a stuck-counter race; the documented fix is one atomic Lua script. The 429 is the FE's `kind:"throttle"` discriminant (D-08), distinct from the paywall.

Output: a registered Lua throttle in `core/redis.py`, a `throttle_gate` FastAPI dependency, and the `POST /readings` route guarded by it — runnable in parallel with Plan 02 (disjoint files).
</objective>

| request body → throttle key | The throttle must key off identity that cannot be forged |
| worker crash → Redis counter | A counter without a TTL is a permanent self-lockout |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
