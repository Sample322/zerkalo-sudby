<objective>
The production safety valve for generation (ADMIN-05). Today `prompt_templates.slug` is UNIQUE and
the seed upserts `ON CONFLICT (slug)`, so there is exactly one row per template and toggling
`is_active=false` on a required template (`system`, `single_card`, `final_summary`, `deck_modifier_*`,
`final_summary`) just makes `PromptEngine._active_template` raise → every reading honest-fails. That
is a kill-switch, not the "create/activate/roll back a bad version" criterion 1 demands.

This plan makes versions **coexist**: multiple `prompt_templates` rows per `slug`, exactly one active
(enforced by a Postgres partial-unique index), and gives the allowlisted operator admin endpoints +
a minimal AdminScreen surface to publish and roll back versions live — no redeploy. The hot-path
engine query (`slug AND is_active` → `scalar_one_or_none`) is deliberately left UNCHANGED; it already
expects exactly one active row per slug, which the new index guarantees.

See 08-RESEARCH.md §1 for the full design + rejected alternative (live text PATCH is not rollback).
</objective>
