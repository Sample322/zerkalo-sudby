Defining the contracts (and the stub files FlowRoot binds to) first eliminates the scavenger-hunt anti-pattern AND the FlowRoot write-conflict, letting Wave 2 run in parallel.
Output: Extended store (with the `reading` slot + `setReading`) + flow root + SDK seam + reading data contract + copy module + onboarding hook + five screen stub files, all under test, with `App.tsx` rendering `<FlowRoot/>` inside `AuthGate`.
