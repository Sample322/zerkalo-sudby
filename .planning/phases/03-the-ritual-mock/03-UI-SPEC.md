# Phase 3 — UI Design Contract

> Visual and interaction contract for the full front-end reading journey (onboarding → selection → ritual prep → flip-reveal → result) against a MOCK reading. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (bespoke premium-dark ritual UI; not a component-library surface) |
| Preset | not applicable |
| Component library | none — Tailwind v4 CSS-first (`@theme` in `frontend/src/index.css`) + raw `motion` primitives. No Radix/Base UI. |
| Icon library | none — atmospheric SVG sigils / glyphs only (`CardArtFallback` pattern). Do NOT introduce an icon font; use inline serif glyphs (`✦`, deck initial) tinted with `var(--deck-accent)`. |
| Font | `ui-serif, Georgia, "Times New Roman", serif` (already set on `body`, `index.css`). One family only. Serif IS the brand voice — never swap to sans for ritual/reveal/result copy. |

**shadcn gate result:** `components.json` not found. shadcn is intentionally NOT initialized for this phase. Rationale: the surface is a bespoke, opinionated ritual experience driven by a per-deck CSS-variable palette system (6 themes in `deckThemes.css`) and `motion` choreography — shadcn's generic primitives would fight the anti-template design-quality bar and the per-deck theming. Registry safety gate: **not applicable** (no registries, no third-party blocks).

**Theming spine (carries into ALL Phase-3 screens):** every screen reads the live per-deck palette via `data-deck=<slug>` on `<html>` (`useDeckTheme`). The four deck vars below are the ONLY source for atmosphere color in onboarding (pre-selection: root default), selection, ritual, reveal, and result. Never hardcode a deck hex in a component.

| CSS var | Role | Default (root) |
|---------|------|----------------|
| `--deck-bg` | Screen background base | `#0b0a12` |
| `--deck-deep` | Gradient deep / glass underlayer | `#1b1726` |
| `--deck-accent` | Accent: active borders, glow, particles, CTA fill, sigils | `#c9a45c` |
| `--deck-soft` | Primary text on dark (titles, card names) | `#e5d3a3` |

Body background is the established radial gradient `radial-gradient(120% 90% at 50% -10%, var(--deck-deep), var(--deck-bg) 70%)` with a 400ms `background`/`color` transition — reuse, do not replace.

## Spacing Scale

Declared values (multiples of 4 — matches Tailwind 4px base already in use: `gap-2/3/6`, `px-4`, `p-4`, `pb-24`):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `1` | Icon/sigil gaps, chip inner rhythm, micro-padding |
| sm | 8px | `2` | Topic-chip gaps, compact stacks, badge padding |
| md | 16px | `4` | Default screen horizontal padding (`px-4`), card inner padding (`p-4`), input padding |
| lg | 24px | `6` | Vertical rhythm between selection sections (`gap-6`), result block spacing |
| xl | 32px | `8` | Major section breaks, onboarding slide vertical centering gaps |
| 2xl | 48px | `12` | Ritual screen vertical breathing room, hero spacing on onboarding |
| 3xl | 64px | `16` | Reserved page-level top spacing on full-bleed ritual/reveal |

**Exceptions (this phase):**
- **Sticky bottom CTA reserve:** scroll regions reserve `pb-24` (96px) so the fixed «Начать расклад» / result CTA never occludes content. 96px is a multiple of 4 (composite of 64 content gap + the CTA band) — keep it.
- **Touch targets:** every tappable (chip, deck tile, spread tile, flip card, in-app back, CTA) must be ≥ 44px in its smallest tap dimension (WCAG 2.5.5 / mobile). Chips currently `px-4 py-2` + text clear 44px; flip cards are 120×192 — fine. This is a min-size floor, not a spacing token.
- **Safe-area padding is additive and dynamic** (see Telegram Adaptation), applied on top of these tokens via SDK insets — never replaces them.

## Typography

Exactly 4 sizes, exactly 2 weights. Serif throughout. Body line-height 1.5, heading line-height 1.2 (display tightest at 1.15).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 600 (semibold) | 1.15 | Onboarding slide title, ritual beat headline, result «Расклад готов» |
| Heading | 20px | 600 (semibold) | 1.2 | Card name on reveal/result, deck title, summary section heads (`text-lg`/`text-xl` band) |
| Body | 16px | 400 (regular) | 1.5 | Onboarding subtitles, card short meaning + interpretation, summary prose, question input text |
| Label | 13px | 400 (regular) | 1.4 | Section eyebrows (uppercase tracked, `opacity-70`), badges, meta row (тема/колода/дата), helper hints |

**Rules:**
- Only these two weights — 400 and 600. No 500/700/light. Hierarchy comes from **size + the 60% `--deck-soft` opacity ladder** (full-opacity titles, `opacity-70` eyebrows, `opacity-60` tertiary), matching the existing catalog.
- Uppercase + `tracking-wide` is reserved for the Label eyebrow role only (e.g. «КОЛОДА СОВЕТУЕТ»). Never uppercase body or card interpretation.
- Italic is allowed only for deck `tone` micro-text and the per-card mystical accent / short phrase (matches `DeckCard` tone styling) — it signals "the deck's voice."
- Card interpretation paragraphs: max ~2–4 short paragraphs (TZ §17), each separated by `lg` (24px). Avoid walls of text (D-01 felt-quality / TZ "избегай слишком длинных абзацев").

## Color

60 / 30 / 10 split expressed through the per-deck vars (NOT fixed hexes — these shift per selected deck). Percentages describe surface dominance on any single screen.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--deck-bg)` + body radial gradient (`--deck-deep`→`--deck-bg`) | Full-screen background on every step; the dark ritual canvas |
| Secondary (30%) | Glass surfaces: `linear-gradient(155deg, color-mix(--deck-bg 88%), color-mix(--deck-deep 72%))` with `1px` border `color-mix(--deck-soft 18–22%, transparent)` | Cards (deck tiles, spread tiles, card-result cards, summary panel, onboarding slide panel, paywall-less sheets) |
| Accent (10%) | `var(--deck-accent)` | See reserved-for list below — strictly rationed |
| Text-primary | `var(--deck-soft)` | Titles, card names, key prose-on-dark (not a "color" in the 60/30/10 sense but the constant on-surface ink) |
| Destructive | NONE this phase | No destructive actions exist in Phase 3 (see Copywriting). Do not introduce a red/error hue; soft errors use the standard `--deck-soft`/accent voice, never an alarm color. |

**Accent (`--deck-accent`) reserved for — explicit list, never "all interactive elements":**
1. The single primary sticky CTA fill/glow («Начать расклад», and the result primary «Ещё расклад»).
2. Active/selected state of a topic chip, deck tile, and spread tile (border + faint `color-mix 10–18%` wash + glow) — exactly as Phase 2 already does.
3. The recommendation banner border + label («Колода советует»).
4. Card edge-glow on flip and the ritual particle field (`opacity` of an accent layer — never `box-shadow` blur animation).
5. Sigil/glyph fill in `CardArt` fallback and onboarding decorative marks.
6. Focus ring (`focus-visible:ring-2` resolves to accent) and the per-card short-phrase emphasis.

Everything else (inactive borders, eyebrows, tertiary text, dividers) uses `--deck-soft` at reduced opacity. Two accents on one element = violation. The accent must read as "the deck is speaking," not as generic UI chrome.

## Screen & Interaction Contracts

State machine: a single Zustand `step` field drives `<AnimatePresence mode="wait">` (D-02). Steps: `onboarding → selection → ritual → reveal → result`. In-app back buttons per screen (D-03), `history[]` backs them. No react-router, no Telegram native BackButton.

### Global animation contract (D-01 / D-10 — measurable, first-class acceptance)
- **Compositor-only**: every animated value is `transform` (`x`/`y`/`scale`/`rotate`/`rotateY`) or `opacity`. **Banned animated props:** `width`, `height`, `top`, `left`, `margin`, `padding`, `box-shadow` blur/size, `background-position`, `border`, `font-size`. Edge-glow = animate `opacity` of a pre-rendered accent layer, never shadow blur.
- **Full animations always** — `<MotionConfig reducedMotion="never">` wraps the flow root. NO `prefers-reduced-motion` downgrade path (accepted tradeoff, Deferred).
- **Bundle discipline**: `<LazyMotion features={domAnimation}>` + `m.*` from `motion/react-m` (ship ~4.6kb not ~34kb). No stray full `motion.*` inside LazyMotion.
- **60fps budget** verified at 360px in the Telegram WebView (Paint-flashing / FPS meter). Particles capped at ~12–16; profile rather than adding a reduced mode.
- **No layout-shift / no pop-in**: preload next-step card art (CSS/SVG fallback mounts warm) during the ritual's ~3s so reveal first-paint is already composited. No content jump at any step boundary.

### Transition durations & easings (locked tokens)
| Transition | Duration | Easing | Notes |
|-----------|----------|--------|-------|
| Screen enter/exit (`AnimatePresence`) | 280ms | `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out-expo) | `initial {opacity:0, y:8}` → `animate {opacity:1, y:0}` → `exit {opacity:0, y:-8}`. `mode="wait"` serializes (no overlap). |
| Deck theme swap (background/color) | 400ms | `ease` | Existing `body` transition — keep; makes deck selection visibly smooth (UI-02). |
| Tap/hover micro-feedback | 150–200ms | default spring / `duration-200` | `whileHover scale 1.03`, `whileTap scale 0.97` (chips 1.05/0.95) — already established. |
| Card flip (`rotateY` 0→180) | spring `stiffness 260, damping 26` | spring | `perspective:1000` on parent, `transformStyle: preserve-3d`, `backfaceVisibility: hidden` on both faces. `onAnimationComplete` → `haptic.impact("light")`. |
| Staggered "раскрыть все" | `delayChildren: stagger(0.12)` | per-card spring (above) | Cascade, never simultaneous pop. |
| Ritual beat crossfade | ~1000ms/beat (~3s total) | `ease` opacity crossfade | Nested `AnimatePresence` keyed on beat index; dimming overlay + particle field underneath. Completion → `haptic.notify("success")` then `goTo("reveal")`. |
| Staggered text reveal (interpretation / summary blocks) | 200–280ms per block, `opacity`+`y` | ease-out-expo | "Появление текста по блокам" (TZ §10.2) — never a hard cut. |

### Screen 1 — Onboarding (ONB-01..04)
- 3–4 full-bleed slides, vertically centered, Display title + Body subtitle, decorative accent sigil. Slide nav = the same `step`-index + `AnimatePresence` primitive (no carousel lib). Swipe OR button («Далее») — both acceptable, constrained by D-01 smoothness.
- «Пропустить» persistent on every slide (ONB-02) → sets localStorage flag + jumps to `selection`.
- Final slide CTA «Сделать первый расклад».
- `onboarding_completed` → localStorage (`zerkalo.onboarding_completed`, D-11); flow root initial step = seen ? `selection` : `onboarding`. Not re-shown (ONB-04).
- Reversed-cards explainer (ONB-03) lives here in plain language (see copy).

### Screen 2 — Selection (extends existing `CatalogScreen`; HOME-01..07)
- Keep the established layout: atmospheric header → **question input (NEW)** → topic chips → recommendation banner → deck carousel → spread list → **sticky «Начать расклад» (NEW)**.
- **Question input (HOME-01/02/D-13):** multiline text field, placeholder «О чём спросим колоду?» (TZ §9.2). Glass surface, Body 16px, `--deck-soft` text, accent focus ring. Render user text as a React text node ONLY — never `dangerouslySetInnerHTML` (V5 security). Validation: empty → valid (general); 1–9 → soft hint; 10–500 → valid; >500 → clamp + hint. **Keyboard safety (UI-04):** size scrollable region to `viewportStableHeight`, keep the field reachable on focus; the iOS keyboard overlaps bottom-fixed elements — never rely on `100vh`/`env()`.
- **«Начать расклад» (HOME-07):** sticky bottom, accent-filled, full-width minus `md` gutters, padded above `--tg-safe-area-inset-bottom`. Enabled only when topic+deck+spread chosen; else disabled + gentle hint. On tap → `createReading(...)` seam (the single Phase-4 boundary) → `goTo("ritual")`.
- Reuse `TopicChip`, `DeckCarousel`/`DeckCard`, `SpreadCard`, recommendation banner verbatim.

### Screen 3 — Ritual prep (READ-07/D-08)
- Auto-advancing ~3s, 3 beats (crossfade Display headline), dimming overlay (animate overlay `opacity`), compositor-only particle field tinted `var(--deck-accent)`.
- Completion haptic `notify("success")`; tap-to-skip after beat 1.
- Preload reveal/result art here (no pop-in at the boundary).

### Screen 4 — Reveal (READ-08/D-09)
- One `FlipCard` per spread position. Each shows: рубашка (card back) → position title → tap to flip → 3D `rotateY` → card name + orientation + short in-character phrase → «Прочитать значение» reveals interpretation block (staggered text).
- After the **first** flip, «Раскрыть все» appears and staggers-flips the rest.
- Card face uses `CardArt` fallback (no art seeded). Card frame 120×192, radius 12 (existing). Edge-glow = accent layer opacity.
- Per-flip haptic `impact("light")`.

### Screen 5 — Result (READ-09/D-12)
- Header «Расклад готов» (Display). Meta row (Label): вопрос / тема / колода / расклад / дата.
- One result card per drawn card (Secondary glass surface): name (Heading), position, orientation, short meaning, interpretation, deck mystical accent (italic).
- Summary panel ("final gather", TZ §10.2): связка карт / главный фактор / на что обратить внимание / мягкий совет / завершающая фраза — revealed as staggered blocks.
- Sticky actions: «Ещё расклад» (accent, wired — D-04) primary; «Сохранить карточку» + «История» secondary stubbed («скоро»).
- All fields populated from the `MockReading` type (schema-faithful to READ-05/06, D-05).

## Telegram Adaptation (UI-04)

- **Theme:** read `colorScheme` / `themeParams` via the `telegram.ts` seam (extend it: `getColorScheme()`, `getThemeParams()`). Default `dark` = premium baseline. The premium-dark deck palette stays dominant in both Telegram light and dark; adapt only chrome/contrast, never abandon the dark ritual canvas (TZ §21.1 "основной стиль лучше тёмный").
- **Safe-area:** use SDK insets — `safeAreaInset` / `contentSafeAreaInset` (Bot API 8.0) via the seam, applied as inline padding or a CSS var you set. The `--tg-safe-area-inset-*` Telegram-provided CSS vars are acceptable. **NEVER** the browser's `env(safe-area-inset-*)` and **never** `100vh`.
- **Sticky CTA** (selection + result): `position` pinned bottom, padded by `--tg-safe-area-inset-bottom`; sized against `viewportStableHeight` so the iOS keyboard cannot hide it.
- **Haptics** (UI-03/READ-07/08): `HapticFeedback` via the seam — `notify("success")` on ritual completion, `impact("light")` per card flip. Optional `selection()` on chip/deck taps. All optional-chained (no-op outside Telegram).

## Anti-Template Compliance (web/design-quality)

This phase must demonstrate ≥4 required qualities and avoid banned generic patterns. Declared intent:
1. **Hierarchy via scale contrast** — 28/20/16/13 ladder + opacity steps, not uniform emphasis.
2. **Intentional rhythm** — `lg` (24px) section rhythm on selection, generous `2xl` breathing on the ritual; not uniform padding.
3. **Depth/layering** — glass gradient surfaces over a radial-gradient canvas, card edge-glow, dimming overlay, 3D flip, particle field.
4. **Typography with character** — serif throughout (the brand voice), italic for the deck's "spoken" lines.
5. **Color used semantically** — accent is rationed to "the deck speaking" (CTA, selection, glow, sigil), never decorative chrome; whole palette shifts per deck so one question genuinely feels different per deck (the Core Value).
6. **Designed hover/focus/active** — `whileHover`/`whileTap` scale, accent focus ring, active selection glow — already established, extended consistently.
7. **Motion that clarifies flow** — ritual beats and staggered reveal pace the journey; AnimatePresence makes steps feel like one continuous ritual, not page loads.

Banned (must NOT appear): default card grid with uniform spacing/no hierarchy; stock centered-headline + gradient-blob hero; unmodified library defaults; flat no-depth layout; safe gray-on-white + one accent; dashboard-by-numbers. Do NOT auto-default to a generic look — the per-deck atmosphere is the point.
