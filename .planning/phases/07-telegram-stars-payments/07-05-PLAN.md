<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ЮKassa → /webhook | Public, unauthenticated, unsigned; IP allowlist is the edge gate, re-fetch (service) is the real guard. |
| client → create/cancel | Authenticated JWT; identity from `sub` only — never a body user_id. |
| admin → refund | `require_admin` server-side allowlist; privileged action. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-WEBHOOK-FORGE | Spoofing | `/payments/yookassa/webhook` | mitigate | IP-allowlist reject before any work + service re-fetch by id; body status never read for granting. |
| T-07-WEBHOOK-DOS | DoS | public webhook | mitigate | Cheap IP-allowlist drop of non-ЮKassa sources before DB/SDK; always-200 on handled stops redelivery storms. |
| T-07-REPLAY | Tampering | redelivered event | mitigate | Always 200 on handled/dup; the service's conditional-UPDATE grant is exactly-once (Plan 03). |
| T-07-IDOR | Elevation | refund / cancel | mitigate | Refund behind `require_admin`; cancel scoped to JWT `sub`, non-owned id → 404 (no existence leak). |
| T-07-AMOUNT | Tampering | create route | mitigate | `CreatePaymentIn` carries only `product_slug`; price recomputed in the service. |
| T-07-SECRET-LEAK | Info Disclosure | responses | mitigate | Routes return only safe projections; `raw_update`/secret never serialized to the client. |
</threat_model>
