<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user question → classify call | The user's free-text question is untrusted input; it is data inside a fixed system frame, never an instruction to the model. |
| user question → generation prompt | Prompt-injection attempts must not redirect the model; structured output + safety pre-classify constrain it. |
| LLM response → backend | The model's output must be schema-validated before any persistence; non-schema output is a failure, not data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-15 | Tampering / Elevation | Prompt injection via the user's question | mitigate | Question is data inside the fixed §16 system frame; `messages.parse` constrains the response to `ReadingOutput`; SafetyService pre-classifies abusive/manipulative (→ redirect). |
| T-04-16 | (Safety/abuse) | Crisis question reaches generation and is served a fatalistic prediction | mitigate | Regex pre-filter catches the highest-signal crisis terms with zero latency + the Haiku classifier covers the rest; `route()` returns REFUSAL — Plan 05 short-circuits BEFORE CardDraw (D-03 locked). |
| T-04-17 | Tampering | Model returns malformed/non-schema JSON | mitigate | `messages.parse` constrained decoding + `parsed_output` ValidationError → tenacity one corrective retry (Sonnet) → reraise (no parse-and-repair; §29.2). |
| T-04-18 | DoS / cost | Classify call cost/latency on every reading | mitigate | Regex pre-filter removes the round-trip for the clearest cases; the classify call is a tiny enum-only Haiku parse (fraction of a cent); logged for audit. |
