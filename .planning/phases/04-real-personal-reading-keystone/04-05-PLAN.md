<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /api/readings | Untrusted body (question, slugs, reversals, possibly a forged user_id/cards): cards + limit + identity are all server-authoritative. |
| safety gate → generation | The crisis short-circuit MUST run before any draw/charge — the most important ordering invariant in the phase. |
| LLM failure → client | A generation crash must surface as a soft in-character 200 body, never a stack trace, and must not consume the limit. |
| generated text → persistence/response | Brand-voice violations must be flagged; card names/orientations are authoritative server-side, not the model's echo. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-23 | Spoofing / Access Control | Client creates a reading as another user / forges user_id | mitigate | `user` derived only from `get_current_user` (JWT sub); body `user_id` ignored; asserted by `test_user_from_jwt_not_body` (V4). |
| T-04-24 | Tampering | Client forges drawn cards / reversals / limit state | mitigate | Backend-only CSPRNG draw (Plan 02) + server-side limit check; `reversals_enabled` read from the validated body only; client cards ignored (§29.2). |
| T-04-25 | (Safety) | Crisis question served a mystical prediction / charged | mitigate | `SafetyService.classify` + `route()` run BEFORE `CardDrawService.draw`; crisis→refusal with NO draw/generation and limit kept (D-03/SAFE-03); asserted by `test_no_cards_on_crisis`. |
| T-04-26 | DoS / cost | Limit consumed on failure or rapid retry abuse | mitigate | Limit consumed in exactly one success-branch place (Pitfall 4); honest-fail keeps it (retry is the same failed reading, not a new charge); the Redis throttle is Phase 6. |
| T-04-27 | Information Disclosure | LLM/stacktrace detail leaks to the client | mitigate | Honest-fail returns the soft §9.8 200 body; truncated `generation_error` stored server-side; the global handler (core/errors.py) backstops unhandled 500s (INFRA-05). |
| T-04-28 | Repudiation / audit | A generation has no audit record | mitigate | One `GenerationLog` row per LLM call (classify + each attempt) with model/tokens/latency/status/prompt_version (ANALYTICS-02). |
| T-04-29 | Tampering | SQL injection in reading/card/log persistence | mitigate | SQLAlchemy 2.0 `select()`/ORM inserts — parameterized, no f-string SQL (mirrors catalog.py). |
</threat_model>
