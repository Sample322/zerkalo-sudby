<objective>
Build the backend half of the Deck & Spread Catalog: three thin auth-gated routers (`GET /api/decks`, `GET /api/decks/{slug}`, `GET /api/spreads`, `GET /api/spreads/recommend`) delegating to a thick `CatalogService`, with Pydantic v2 `from_attributes` response schemas; add the missing `SpreadType.positions` ORM relationship; and close the SPREAD-04 seed gap by deriving + seeding `deck_spread_compatibility` from REFERENCE-TZ §7.

This is the data + API slice of the vertical feature. After this plan, an authenticated client can fetch the full catalog and a topic-aware recommendation over HTTP — the frontend slice (Plan 03) consumes exactly these endpoints.

Purpose: Serve the seeded Phase 1 catalog data (6 decks, 7 spreads, 23 positions) and make SPREAD-04's "honor compatibility" claim real (the table is currently empty).
Output: 2 model edits, 1 seed-data file, 1 loader extension, 1 schemas module, 1 service module, 2 routers, main.py wiring, 4 test files.
</objective>
