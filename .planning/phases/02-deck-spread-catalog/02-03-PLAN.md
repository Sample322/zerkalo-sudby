<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| React client → FastAPI catalog | All catalog fetches go through `apiFetch` (Bearer JWT from Phase 1 session store) |
| API JSON → React render | React escapes by default; query params built with URLSearchParams |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | Catalog fetches | mitigate | All requests use `apiFetch`, which attaches the Phase 1 Bearer JWT; unauthenticated users never reach CatalogScreen (rendered inside AuthGate's authenticated state). |
| T-02-02 | Tampering | recommend/spreads query params | mitigate | `URLSearchParams` builds the query string (no manual concatenation of user-influenced values); backend (Plan 01) validates + parameterizes. |
| T-02-07 | Tampering (XSS) | Rendering deck/spread/reason text | mitigate | React auto-escapes all rendered API text; no `dangerouslySetInnerHTML` anywhere in this plan. |
| T-02-05 | Information Disclosure | Recommendation reason + all catalog copy | mitigate (product/compliance) | Reason is rendered from the API (Plan 01 already brand-voice-tested); CatalogScreen adds no AI/нейросеть/модель strings — asserted by the reason scan in CatalogScreen.test.tsx + grep. |
| T-02-09 | Information Disclosure | Catalog error state | mitigate (inherited) | Error/loading/empty states render in-character (mirror AuthGate); no stack trace, exception class, or internal detail surfaced to the WebView. |
| T-02-SC | Tampering | npm installs | n/a | This plan installs ZERO new packages (uses deps already present + RTL from Plan 02). No legitimacy gate needed. |
</threat_model>
