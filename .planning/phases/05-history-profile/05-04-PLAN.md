| Boundary | Description |
|----------|-------------|
| client → API | Bearer JWT is the only identity; `{reading_id}` is a UUID path (422 on malformed); no body identity on any verb. |
| API → DB | All queries are `select()`-parameterized + user-scoped; soft delete is a timestamp write, never a hard delete. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-05-IDOR-GET | Information Disclosure / Elevation | `GET /api/readings/{id}` | mitigate | `where(id, user_id == user.id, deleted_at IS NULL)`; non-owned → `ReadingInputError` → 404 (not 403 — no existence leak). `test_cross_user_detail_and_delete_404`. |
| T-05-IDOR-DEL | Tampering | `DELETE`/`restore /api/readings/{id}` | mitigate | Same user-scoped `where` on delete + restore; a non-owned id → 404; one user cannot restore another's row. |
