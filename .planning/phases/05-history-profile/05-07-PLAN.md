<objective>
Ship the Profile/Settings screen and migrate the onboarding flag to the server. Profile shows the Telegram identity (name + photo from `GET /api/me`) + the two toggles (reversals, history-personalization) — NO stats, NO readings-count/subscription block (hidden in Phase 5 per D-08). All new copy strings already live in copy.ts (05-05) — this plan imports them and does NOT edit copy.ts.
Output: `me.ts` client + `useMe`/`usePatchSettings` hooks, the Profile screen, the FlowRoot onboarding-gate change, the CatalogScreen reversals-source change, and the Profile component test.
